ReignX 内置 PXE（Preboot Execution Environment）服务，支持通过网络批量部署操作系统， 无需手动插入光盘或 U 盘，实现零接触自动化装机。

装机流程

1
管理员配置 PXE 服务器：启动 DHCP/TFTP，上传 OS 镜像
2
创建自动化安装配置文件（Kickstart/Preseed/Cloud-Init）
3
选择目标服务器，通过 Web/CLI/API 发起 OS 安装任务
4
ReignX 通过 IPMI 设置服务器从网络启动（PXE Boot）
5
服务器开机，通过 DHCP 获取 IP 地址和引导文件位置
6
下载 pxelinux.0/grub 引导加载器和 Linux 内核/initrd
7
启动安装程序，下载 Kickstart/Preseed 配置文件
8
自动分区、安装软件包、配置网络、设置 root 密码
9
安装完成后执行 Post-Install 脚本（部署 Agent、配置监控）
10
服务器重启，Agent 自动注册到 ReignX 平台
11
状态变更：discovered → installing → online，开始接受任务

ReignX 支持多种方式自动发现和注册服务器节点，无需手动逐台添加，大幅提升大规模集群的部署效率。

Agent 注册流程

1
Agent 启动，加载配置文件（服务器地址、TLS 证书等）
2
建立 gRPC 连接（支持 TLS/mTLS），调用 Register RPC
3
发送节点信息：主机名、IP、OS、CPU、内存、磁盘等
4
API 服务器验证并在数据库中创建/更新服务器记录
5
Agent 启动心跳协程，每 30 秒发送 Heartbeat 请求
6
心跳包含系统指标（CPU/内存/磁盘使用率、运行时间）
7
服务器更新 last_seen 时间戳，指标持久化到数据库
8
HeartbeatMonitor 检测超时节点，标记为 offline

节点发现方式对比

发现方式 原理 优势 适用场景
Agent 主动注册 Agent 启动后主动连接 API Server 最可靠，信息最完整 已安装 OS 的服务器
IPMI 扫描 扫描网段 IPMI 端口（623） 无需 OS，可远程管理 裸金属服务器初始化
网段扫描 SSH 端口扫描 + 指纹识别 快速批量发现 已有 SSH 访问权限
DHCP 监听 监听 DHCP 请求获取 MAC 地址 实时发现新设备 PXE 装机环境
云平台 API 调用云厂商 API 同步实例 自动同步元数据 混合云环境

发现方式详解

IPMI 扫描流程

  1. 扫描指定网段的 623 端口
  2. 检测 IPMI 服务响应
  3. 使用默认凭据尝试登录
  4. 读取主板序列号和型号
  5. 获取电源状态和传感器数据
  6. 导入到 ReignX 数据库

SSH 扫描流程

  1. 并发扫描网段的 22 端口
  2. 检测 SSH 服务 Banner
  3. 尝试使用 SSH Key 认证
  4. 执行 uname、dmidecode 收集信息
  5. 读取 CPU/内存/磁盘配置
  6. 自动创建服务器记录

⚖️ Agent vs SSH 模式对比

特性 Agent 模式 SSH 模式
通信协议 gRPC (高性能二进制) SSH (文本协议)
性能 ⚡⚡⚡ 优秀（持久连接） ⚡⚡ 良好（每次建连）
实时性 ✓ 实时推送任务 ✗ 需主动轮询
资源占用 ~20MB 内存（Agent 进程） 0（无需安装）
部署复杂度 需部署 Agent 二进制 仅需 SSH 访问权限
防火墙友好 需开放 gRPC 端口 ✓ 仅需 SSH (22)
认证方式 mTLS + Token SSH Key / Password
适用场景 大规模集群、实时管理 快速接入、临时管理
故障恢复 自动重连 + 指数退避 按需连接
模式切换 ✓ 支持无缝切换

推荐使用场景

🚀 Agent 模式

  • 生产环境大规模服务器集群
  • 需要实时监控和告警
  • 频繁的批量操作
  • 对性能有较高要求
  • 需要流式任务推送

🔧 SSH 模式

  • 快速接入新服务器
  • 临时或测试环境
  • 受限网络环境（仅 SSH）
  • 不便安装 Agent 的场景
  • 偶尔需要执行命令

📚 技术参考

数据库设计和 REST API 端点参考，方便开发和集成。

核心表结构

-- 服务器表
CREATE TABLE nodes (
    id UUID PRIMARY KEY,
    hostname VARCHAR(255) UNIQUE NOT NULL,
    ip_address VARCHAR(45),
    status VARCHAR(50),                  -- online, offline, installing
    mode VARCHAR(20),                    -- agent, ssh, hybrid
    os_type VARCHAR(50),
    cpu_cores INTEGER,
    memory_gb INTEGER,
    metadata JSONB,                      -- 扩展字段（指标、标签等）
    last_seen TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 任务表
CREATE TABLE jobs (
    id UUID PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    type VARCHAR(50),                    -- command, script, package
    status VARCHAR(50),                  -- pending, running, completed
    filter JSONB,                        -- 服务器筛选条件
    template JSONB,                      -- 任务模板
    batch_size INTEGER,
    concurrency INTEGER,
    completed INTEGER DEFAULT 0,
    failed INTEGER DEFAULT 0,
    created_by VARCHAR(255),
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- SSH 配置表（加密存储）
CREATE TABLE ssh_configs (
    id UUID PRIMARY KEY,
    node_id UUID REFERENCES nodes(id),
    username VARCHAR(255),
    password TEXT,                       -- AES-256 加密
    host_key_fingerprint VARCHAR(255),   -- SHA256 指纹
    known_hosts_entry TEXT,
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 会话录制表
CREATE TABLE session_recordings (
    id UUID PRIMARY KEY,
    session_id VARCHAR(255) UNIQUE,
    user_id VARCHAR(255),
    server_id UUID REFERENCES nodes(id),
    started_at TIMESTAMPTZ,
    ended_at TIMESTAMPTZ,
    recording_path TEXT,                 -- asciinema 格式文件路径
    playback_format VARCHAR(50)
);

索引策略

  • nodes: idx_nodes_status, idx_nodes_last_seen, idx_nodes_hostname
  • jobs: idx_jobs_status, idx_jobs_created_at, idx_jobs_created_by
  • tasks: idx_tasks_job_id, idx_tasks_node_id, idx_tasks_status
  • ssh_configs: unique_idx_node_id, idx_host_key_fingerprint
  • session_recordings: idx_user_id, idx_server_id, idx_started_at

🔒 安全特性

🔐 认证机制

  • JWT 双 Token 机制（Access + Refresh）
  • Access Token 15 分钟过期
  • Refresh Token 7 天过期
  • bcrypt 密码哈希（成本因子 10）
  • mTLS 双向认证（Agent 模式）

🔑 数据加密

  • AES-256 GCM 对称加密
  • SSH 密码加密存储
  • Base64 编码存储密文
  • 环境变量管理加密密钥
  • TLS 1.2+ 传输加密

🛡️ SSH 安全

  • SSH 主机密钥验证
  • SHA256 指纹存储
  • 防中间人攻击（MITM）
  • 公钥认证优先
  • SSH Agent 集成（macOS Keychain）

📝 审计日志

  • 所有终端会话录制
  • asciinema 格式可回放
  • API 请求日志（结构化）
  • 用户操作审计
  • 安全事件告警

完整操作示例

# 1. 发现网段内的服务器
reignxctl discover ssh --network 192.168.1.0/24 --ssh-key ~/.ssh/id_rsa

# 2. 查看已发现的服务器
reignxctl server list

# 3. 批量执行命令
reignxctl batch exec --filter "status=online" --command "uptime"

# 4. 创建批量更新任务
reignxctl job create --name "System update" \
  --type command \
  --command "apt update && apt upgrade -y" \
  --filter "os_type=ubuntu" \
  --concurrency 20

# 5. 监控任务进度
reignxctl job get job-12345 --watch

# 6. 上传配置文件到所有 Web 服务器
reignxctl batch upload --filter "role=webserver" \
  --local ./nginx.conf \
  --remote /etc/nginx/nginx.conf

# 7. 批量重启 Nginx
reignxctl batch exec --filter "role=webserver" \
  --command "nginx -t && systemctl restart nginx" \
  --batch-size 5
