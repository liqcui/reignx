# ReignX Deployment Guide

This directory contains deployment configurations for the ReignX distributed server management platform.

## Deployment Options

### 1. Local Development (Docker Compose)

**Quick Start:**
```bash
# Build and deploy
./deployments/scripts/deploy-dev.sh

# Access services
open http://localhost:8080      # ReignX API
open http://localhost:3000      # Grafana (admin/admin)
open http://localhost:9090      # Prometheus
open http://localhost:16686     # Jaeger

# View logs
docker-compose -f deployments/docker-compose/docker-compose.dev.yaml logs -f

# Stop services
docker-compose -f deployments/docker-compose/docker-compose.dev.yaml down
```

**Components:**
- PostgreSQL (port 5432)
- etcd (port 2379)
- NATS JetStream (port 4222)
- ReignX Daemon (ports 8080, 9000)
- Prometheus (port 9090)
- Grafana (port 3000)
- Jaeger (port 16686)
- Redis (port 6379)

### 2. Kubernetes Production Deployment

**Prerequisites:**
- Kubernetes 1.25+
- kubectl configured
- Storage class for PersistentVolumes
- LoadBalancer support (or Ingress controller)

**Quick Deploy:**
```bash
# Deploy to Kubernetes
./deployments/scripts/deploy-k8s.sh

# Or use Kustomize
kubectl apply -k deployments/kubernetes/

# Access services via port-forward
kubectl port-forward -n reignx svc/reignxd-service 8080:8080
kubectl port-forward -n reignx svc/grafana-service 3000:3000
```

**Production Checklist:**
- [ ] Update secrets in `kubernetes/secrets.yaml`
- [ ] Generate real TLS certificates using `scripts/init-pki.sh`
- [ ] Configure persistent storage class
- [ ] Set up Ingress for external access
- [ ] Configure backup strategy for PostgreSQL
- [ ] Review resource limits and HPA settings
- [ ] Set up monitoring alerts

### 3. Manual Docker Builds

**Build all images:**
```bash
./deployments/scripts/build-images.sh
```

**Build individual images:**
```bash
# ReignX Daemon
docker build -f deployments/docker/Dockerfile.reignxd -t reignx/reignxd:latest .

# Agent
docker build -f deployments/docker/Dockerfile.agent -t reignx/reignx-agent:latest .

# CLI
docker build -f deployments/docker/Dockerfile.cli -t reignx/reignxctl:latest .

# Migrate
docker build -f deployments/docker/Dockerfile.migrate -t reignx/migrate:latest .
```

   ┌────▼────┐              ┌─────▼──────┐

## Security

### TLS/mTLS Setup

1. **Generate certificates:**
```bash
./scripts/init-pki.sh
```

2. **Create Kubernetes secret:**
```bash
kubectl create secret generic reignx-certs \
  --from-file=ca-cert.pem=certs/ca/root/ca-cert.pem \
  --from-file=server-cert.pem=certs/server/apiserver-cert.pem \
  --from-file=server-key.pem=certs/server/apiserver-key.pem \
  -n reignx
```

3. **Certificate rotation:**
- Certificates expire after 365 days (configurable)
- Auto-renewal happens when <30 days remaining
- Manual renewal: Re-run `init-pki.sh` and update secrets

### Secrets Management

**Development:**
- Secrets stored in `kubernetes/secrets.yaml` (Git-ignored in production)

**Production:**
- Use Kubernetes Secrets with encryption at rest
- Or external secret managers:
  - HashiCorp Vault
  - AWS Secrets Manager
  - Azure Key Vault
  - Google Secret Manager

## Backup & Recovery

### PostgreSQL Backup

**Manual backup:**
```bash
kubectl exec -n reignx postgres-0 -- \
  pg_dump -U bm_admin bm_solution > backup.sql
```

**Automated backups:**
- Use `pg_basebackup` for continuous archiving
- Or PostgreSQL operators (Zalando, Crunchy Data)
- Or cloud-native solutions (RDS snapshots, Cloud SQL)

### etcd Backup

**Snapshot:**
```bash
kubectl exec -n reignx etcd-0 -- \
  etcdctl snapshot save /tmp/snapshot.db
```

## Troubleshooting

### Common Issues

**ReignXD pods not starting:**
```bash
# Check logs
kubectl logs -n reignx -l app=reignxd

# Check events
kubectl get events -n reignx --sort-by='.lastTimestamp'

# Verify database connectivity
kubectl exec -n reignx reignxd-xxx -- nc -zv postgres-service 5432
```

**Database connection errors:**
```bash
# Check PostgreSQL status
kubectl exec -n reignx postgres-0 -- pg_isready

# Check credentials
kubectl get secret postgres-secret -n reignx -o yaml
```

**etcd cluster issues:**
```bash
# Check cluster health
kubectl exec -n reignx etcd-0 -- etcdctl endpoint health

# Check member list
kubectl exec -n reignx etcd-0 -- etcdctl member list
```

### Performance Tuning

**Database:**
- Increase `max_connections` for high load
- Tune `shared_buffers` (25% of RAM)
- Enable connection pooling (PgBouncer)

**ReignXD:**
- Adjust HPA thresholds based on load patterns
- Increase CPU/memory limits for large deployments
- Enable HTTP/2 for gRPC connections

## Upgrade Strategy

### Rolling Update

```bash
# Update image version
kubectl set image deployment/reignxd \
  reignxd=reignx/reignxd:v1.1.0 \
  -n reignx

# Monitor rollout
kubectl rollout status deployment/reignxd -n reignx

# Rollback if needed
kubectl rollout undo deployment/reignxd -n reignx
```

### Database Migration

```bash
# Run migrations
kubectl run migrate --rm -i --tty \
  --image=reignx/migrate:latest \
  --env="DATABASE_URL=$DB_URL" \
  -- up
```

## Cost Optimization

**Development:**
- Use Docker Compose (single machine)
- Disable monitoring stack if not needed
- Use lightweight storage (emptyDir)

**Production:**
- Use spot instances for non-critical workloads
- Enable cluster autoscaling
- Set appropriate resource requests/limits
- Use reserved instances for stable components
